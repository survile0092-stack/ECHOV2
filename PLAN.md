# Исправить риски потери данных и мелкие баги

## Что будет сделано

### 🛡️ Надёжное сохранение фото домиков

- Если копирование фото не удалось обычным способом, приложение автоматически попробует альтернативный путь (через скачивание во внутреннюю папку), а не оставит временную ссылку.
- В хранилище будут сохраняться только имена файлов — никогда абсолютные пути. Это гарантирует, что фото не пропадут после обновления приложения или iOS/Android.
- Если фото всё-таки не удалось сохранить навсегда — пользователь увидит понятное сообщение об ошибке вместо «битой» картинки потом.

### 🔄 Защита данных при обновлениях приложения

- При запуске приложение будет «доводить» старые записи до актуального формата: добавлять недостающие поля у бронирований и домиков (предоплата, остаток, кол-во гостей, цены, дни уведомлений и т.д.).
- Чтение хранилища будет защищено от повреждённых данных — если файл повреждён, приложение не упадёт, а покажет понятное сообщение и продолжит работу.
- Добавится номер версии схемы, чтобы будущие обновления могли безопасно мигрировать данные.

### 📥 Безопасный импорт бэкапа

- При импорте бэкапа всегда будет показываться диалог: сколько домиков и бронирований будет добавлено/заменено, и предложение выбрать «Заменить всё» или «Объединить с текущими» (объединение по ID — новые добавляются, существующие обновляются).
- Будет проверка версии файла бэкапа — если версия новее, чем поддерживает приложение, появится предупреждение.

### 🧹 Очистка фото при удалении домика

- Когда домик удаляется, его фотографии тоже будут удаляться с устройства, чтобы не накапливался мусор и не занимал память.

### 🐞 Мелкие баги

- Если пользователь убрал все дни уведомлений в настройках — уведомлений действительно не будет (сейчас они тайно возвращаются).
- Кол-во гостей по умолчанию будет одинаковым везде (в аналитике и в Excel-экспорте).
- Убраны неиспользуемые куски кода в экране создания брони.

## Что не меняется

- Внешний вид экранов остаётся прежним.
- Все существующие данные сохраняются и автоматически приводятся к новому формату при первом запуске.
- Бронирование без имени гостя по-прежнему возможно (плейсхолдер «Имя гостя не указано» с жёлтой подсветкой остаётся).
